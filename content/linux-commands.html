Linux Commands Basics

Linux commands are powerful tools for interacting with your system, performing tasks efficiently, and automating processes. Below are some of the basic Linux commands that every user should know:

  1. pwd: Displays the present working directory
  2. clear: Used to clear the terminal
  3. cd <path>: Used to change the current working directory
  4. cd: Changes the directory to the home directory
  5. cd ~: Changes the directory to the home directory
  6. cd /: Changes the directory to the root directory
  7. cd ..: Changes the directory to its parent directory
  8. cd 'folder name': Used to change the directory when there is a space in the directory name
  9. ls: Lists all the contents in the current working directory
  10. ls <path>: By specifying the path after ls, the content in that path will be displayed
  11. ls -l: Lists all the contents along with their owner, permissions, size, and timestamp (long format)
  12. ls -a: Lists all the hidden contents in the specified directory
  13. ls -author: Lists the contents in the specified directory along with their owner
  14. ls -lS /path/to/folder/: Lists all the contents in the specified directory by size
  15. ls *.<format>: Lists only the contents in the directory of a particular format
  16. ls -lS > file: Copies the result of ls command into a text file
  17. cat <file>: Used to display the content of the file on terminal
  18. cat -b <file>: Used to add line numbers to non-blank lines
  19. cat -n <file>: Used to add line numbers to all lines
  20. cat -s <file>: Used to squeeze blank lines into one line
  21. cat -E <file>: Show $ at the end of each line
  22. cat > file: Used to create a new file and enter text contents from the terminal
  23. cat >> file: Used to append text contents to an existing file from the terminal
  24. grep <string> file: Returns results for matching string
  25. grep -i <string> file: Returns results for case insensitive string
  26. grep -n <string> file: Returns the matching string along with their line number
  27. grep -v <string> file: Returns the result of lines not matching the string
  28. grep -c <string> file: Returns the number of lines which matches the string
  29. sort <file>: Sorts the contents of file and returns them in alphabetical order
  30. sort -r <file>: Returns the results in reverse order (descending order)
  31. sort -f <file>: It will do the case insensitive sorting
  32. sort -n <file>: Returns the results as per numerical order
  33. grep <string> <file> | sort: Searches for string and sorts the results
  34. grep <string> <file> | sort -r: Sorts the results in reverse order
  35. cp <file1> <file2>: Used to copy files
  36. cp -i <file1> <file2>: Enters interactive mode; CLI asks before overwriting files
  37. cp -n <file1> <file2>: Does not overwrite the file
  38. cp -u <file1> <file2>: Updates the destination file only when source file is different from destination file
  39. cp -R <dir1> <dir2>: Used to copy directories
  40. cp -v <file1> <file2>: Verbose; prints informative messages
  41. mv <file1> <file2>: Used to move and rename files
  42. mv -i <file1> <file2>: Interactive mode; CLI asks before overwriting files
  43. mv -u <file1> <file2>: Updates the destination file only when source file is different from destination file
  44. mv -v <file1> <file2>: Verbose; prints source and destination files
  45. mkdir <directory>: Used to create a directory
  46. mkdir -p file1/file2/file3: Creates both a new parent directory and subdirectory
  47. mkdir -p file1/{f2,f3,f4}: Used to create multiple subdirectories inside the new parent directory
  48. rmdir <dir>: Used to remove the directory (empty)
  49. rmdir -p f1/f2/f3: Used to remove both the parent and child directory (empty)
  50. rmdir -pv f1/f2/f3: Used to remove all the parent and subdirectories along with the verbose
  51. rm -r <dir>: Used to remove even non-empty directories
  52. rm -rp f1/f2/f3: Removes non-empty directories including parent and subdirectories
  53. ls -R <path>: Lists all the contents in specified folder in tree format
  54. less <file>: Used to view the file on terminal (especially for big files) and to move forward or backward inside the file using keyboard keys
  55. head <file>: Used to display the first 10 lines of the file
  56. head -n N <file>: Used to display the first N lines of the file
  57. tail <file>: Used to display the last 10 lines of the file
  58. tail -n N <file>: Used to display the last N lines of the file
  59. find: Used to find the files in the system and it will search for the directories as well
  60. find . -name <file name>: Searches for a file by its name from the current directory
  61. find / -name <file name>: Searches for a file by its name in the root directory
  62. find . -name "*.py": Finds the file based on the extension (Python files)
  63. find . -type f -name "*.py": Finds all the python files in the current directory
  64. find . -type d -name "*.py": Finds all the python directories in the current directory
  65. df: Used to display the disk space usage of the file system
  66. df -h: Used to display the disk space usage in a readable format
  67. du: Used to display the disk space usage of files and directories
  68. du -h: Used to display the disk space usage in a readable format
  69. df -h: Used to display the disk space usage in a readable format
  70. df -a: Used to display all the file systems along with the usage
  71. df -T: Used to display the file system type
  72. man: Used to view the manual pages for the specified command
  73. man ls: Used to view the manual pages for ls command
  74. man cat: Used to view the manual pages for cat command
  75. history: Used to display the list of previously executed commands
  76. history -c: Clears the history
  77. uname: Used to display the system information
  78. uname -r: Used to display the kernel version of the system
  79. uname -a: Used to display all the system information
  80. whoami: Used to display the currently logged-in user
  81. hostname: Used to display the system hostname
  82. hostname -I: Used to display the system IP address
  83. top: Used to display the running processes and resource usage in real-time
  84. ps: Used to display the currently running processes
  85. ps -aux: Used to display detailed information about all running processes
  86. kill: Used to terminate a process by its PID
  87. kill -9 <PID>: Forcefully terminates a process by its PID
  88. ping: Used to test the network connectivity between the system and a specified host
  89. ping google.com: Tests the network connectivity to google.com
  90. wget <url>: Used to download files from the specified URL
  91. curl <url>: Used to transfer data from or to a server using various protocols
  92. wc <file>: Used to display the number of lines, words and characters the file has
  93. wc -l <file>: Used to display the number of lines only (use '-w' flag for number of words and '-c' flag for number of characters)
  94. cal: Used to display the current month
  95. cal <year>: Used to display the calendar of a particular year
  96. cal <month> <year>: Used to display the calendar of a particular month
  97. date: Used to display the date
  98. sudo date -s "01 jan 2050 12:00:00": Used to set the date and time in the system
  99. watch <command>: Used to run the command repeatedly at regular intervals
  100. watch -n <time(s)> <command>: Same as watch but allows setting the time interval
  101. sudo useradd <username> -m: Used to add a new user along with a new home directory (-m)
  102. sudo passwd <username>: Used to set a password for the user
  103. sudo userdel <username>: Used to delete the user
  104. sudo userdel -r <username>: Used to delete the user and the user's home directory
  105. sudo rm -r /home/<username>/: Used to delete the home directory of the user
  106. groups: Used to show the groups the current user belongs to
  107. cat /etc/group/: Used to show all the groups in the system
  108. sudo groupadd <groupname>: Used to add a group
  109. sudo groupdel <groupname>: Used to delete a group
  110. sudo gpasswd -a <username> <groupname>: Used to add the user to a particular group
  111. sudo gpasswd -d <username> <groupname>: Used to remove the user from a particular group
