Instructions for Configuring and Deploying the Invitation Module UI

Posted on May 30, 2024

Configuration Deployment UI

Below are step-by-step instructions for configuring and deploying the Invitation Module UI.

  1. Clone the Repository:

    Navigate to this link and select the main branch. Clone the repository to your local system using the appropriate git command.

  2. Update Configuration:

    Go to the project folder and navigate to ims-ui/src/. Open the Globals.js file. Modify the baseURL variable to reflect the IP address and port number where the invitation module services are deployed. Save the file after making the changes.

  3. Build the Project:

    Open a terminal or command prompt in the root directory of the project (ims-ui/). Execute the following command to build the React project: 

    npm run build

    Upon successful completion, a build folder will be generated in the root directory of the project.

  4. Prepare for Deployment:

    Zip the build folder. Transfer the zipped build folder to the server where you intend to deploy the UI.

  5. Deploy the UI:

    Navigate to the directory where you copied the zipped build folder on the server. Execute the following commands to deploy the UI: 

    # Unzip the build folder
sudo unzip build.zip
# Set appropriate permissions for the build folder
sudo chmod -R 777 build
# Deploy the application using PM2
sudo pm2 serve build 3000

By following these steps, you will be able to successfully configure and deploy the Invitation Module UI.

Instructions for Configuring and Deploying the Invitation Module Services

Posted on May 30, 2024

Configuration Deployment Services

Below are step-by-step instructions for configuring and deploying the Invitation Module Services.

  1. Clone the Repository:

    Navigate to this link and select the main branch. Clone the repository to your local system using the appropriate git command.

  2. Review Source Code and Configurations:

    Navigate to the ims-service/ims-invitation-service/ directory, which contains the source code for the Invitation Module services. Note that all configurations related to the invitation module services are externalized. The application.properties file, SMS & Email Templates, Pass Designs, and Reports Configurations are read from outside the project folder.

  3. Prepare Configuration Files:

    Before building the JAR file and deploying the services, copy all necessary configurations from the local system to the server.

  4. Set Up Server Directory for external config:

    On the server where you intend to deploy the invitation module services, create an empty folder. For this example, create a folder named IMS under the /opt/ directory (/opt/IMS/).

  5. Transfer Configuration Files:

    Copy the following configurations and files from your local system to the server:

    • Copy the ims-service/ims-invitation-service/src/main/resources/application.properties file to the /opt/IMS/ directory on the server.
    • Copy the entire ims-service/ims-invitation-service/src/main/resources folder to the /opt/IMS/ directory on the server.
    • Navigate to the /opt/IMS/resources/ directory on the server and create an empty folder named doc_upload. This folder will store all reports generated by the application.

  6. Update Configuration Properties:

    Open the application.properties file located in the /opt/IMS/ directory on the server, and make the following modifications:

    • Update the database connection details.
    • Adjust the jwtExpirationMs and jwtExpirationMsHighRoles variables.
    • Set the allowedHost variable.
    • Update the template IDs for SMS and Email templates.
    • Set smsEnabled to true to enable real-time SMS and email notifications in the production environment.
    • Modify RESOURCE_LOCATION and DOC_UPLOAD_LOCATION.
    • Comment out the line containing spring.config.import.
    • Update the DOWNLOAD_REQUEST_URL and DOWNLOAD_FILE_URL with the IP and port where the invitation module services will be deployed.

    Save the file after making these changes.

  7. Configure Local Application Properties:

    Open the application.properties file in your local system and set the spring.config.import variable to the external file path of application.properties, which is file:/opt/IMS/application.properties.

  8. Build the Application:

    Navigate to the root directory of the invitation module services (ims-service/ims-invitation-service/) on your local system. Open a terminal or command prompt in this directory and run the following command to build the project: 

    mvn clean package

    This will generate a JAR file in the target folder of the application (ims-service/ims-invitation-service/target).

  9. Deploy the JAR File:

    Transfer the JAR file to the server where you intend to deploy the services. Navigate to the directory where you copied the JAR file and execute the following command to deploy it: 

    sudo nohup java -server -XX:NewSize=512M -Xms2G -Xmx2G -XX:InitialCodeCacheSize=500m -XX:ReservedCodeCacheSize=512M -jar <jarFileName> --server.port=<servicePort> > nohup.out 2>&1 &

By following these instructions, you will be able to successfully configure and deploy the Invitation Module Services.
