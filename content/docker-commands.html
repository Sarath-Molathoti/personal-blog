Docker Commands Basics

Docker commands are essential for managing Docker containers, images, and networks. Below are some of the basic Docker commands that every user should know:

1. Pull an Image

The docker pull command is used to download a Docker image from Docker Hub or another registry. 

docker pull IMAGE_NAME[:TAG]

2. List Downloaded Images

The docker images command lists all downloaded images on your local machine. 

docker images

3. Run a Container

The docker run command creates and starts a new container based on the specified image. 

docker run IMAGE_NAME[:TAG]

4. List Running Containers

The docker ps command lists all running containers.

docker ps

5. List All Containers (including stopped ones)

The docker ps -a command lists all containers, including those that have stopped. 

docker ps -a

6. Stop a Running Container

The docker stop command stops a running container.

docker stop CONTAINER_ID or CONTAINER_NAME

7. Remove a Container

The docker rm command removes a stopped container.

docker rm CONTAINER_ID or CONTAINER_NAME

8. Interactive Shell in a Container

The docker exec command opens an interactive shell in a running container. 

docker exec -it CONTAINER_ID or CONTAINER_NAME /bin/bash

9. Expose Container Port to Host

The docker run -p command maps a port from the container to a port on the host machine. 

docker run -p HOST_PORT:CONTAINER_PORT IMAGE_NAME[:TAG]

10. Volume Mounting

The docker run -v command mounts a volume from the host to the container, allowing data persistence. 

docker run -v HOST_PATH:CONTAINER_PATH IMAGE_NAME[:TAG]

11. Inspect Container Details

The docker inspect command displays detailed information about a container. 

docker inspect CONTAINER_ID or CONTAINER_NAME

12. View Container Logs

The docker logs command displays the logs of a running or stopped container. 

docker logs CONTAINER_ID or CONTAINER_NAME

13. Remove an Image

The docker rmi command removes a Docker image.

docker rmi IMAGE_ID or IMAGE_NAME[:TAG]

14. Network Commands

The docker network ls command lists Docker networks, and docker network inspect inspects a network. 

docker network ls
docker network inspect NETWORK_ID or NETWORK_NAME

15. Remove an Image

The docker rmi command deletes a Docker image.

docker rmi IMAGE_ID

16. Run Container in Detached Mode

The docker run -d command runs a Docker container in detached mode, exposes a port to the host, and gives it a custom name. 

docker run -d -p HOST_PORT:CONTAINER_PORT --name CUSTOM_NAME IMAGE_NAME

17. Access Container Terminal

The docker exec -it command allows access to the terminal of a running Docker container. 

docker exec -it CONTAINER_ID or CONTAINER_NAME /bin/bash

These commands are essential for managing Docker environments effectively. Mastering them will enable you to handle containers, images, and networks with ease.
