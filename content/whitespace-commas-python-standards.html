Whitespace and Commas - Python Coding Standards (Part2)

Posted on May 14, 2024

Python Whitespace Commas Best Practices

In coding, maintaining consistent and clean whitespace usage is essential for code readability and aesthetics. Avoid these common whitespace issues:

2.2 Whitespace and Commas within Expressions and Statements

2.2.1 Whitespace Inside Parentheses, Brackets, or Braces

Avoid extraneous whitespace inside parentheses, brackets, or braces. 

# Correct:
function_call(argument1, [item1, item2])

# Incorrect:
function_call( argument1, [ item1, item2 ] )

2.2.2 Whitespace After Trailing Comma

Avoid placing whitespace after a trailing comma.

# Correct:
foo = (0,)

# Incorrect:
bar = (0, )

2.2.3 Whitespace Before Comma, Semicolon, or Colon

Avoid placing whitespace before a comma, semicolon, or colon.

# Correct:
if condition1: action1(); condition2, condition3 = result1, result2

# Incorrect:
if condition1 : action1() ; condition2 , condition3 = result1 , result2

2.2.4 Whitespace in Slice Notation

For slices, the colon should have equal amounts of spacing on both sides, except when a slice parameter is omitted. 

# Correct:
ham[1:9], ham[1:9:3], ham[:9:3], ham[1::3], ham[1:9:]
ham[lower:upper], ham[lower:upper:], ham[lower::step]
ham[lower+offset : upper+offset]
ham[: upper_fn(x) : step_fn(x)], ham[:: step_fn(x)]
ham[lower + offset : upper + offset]

# Incorrect:
ham[lower + offset:upper + offset]
ham[1: 9], ham[1 :9], ham[1:9 :3]
ham[lower : : step]
ham[ : upper]

2.2.5 Whitespace Before Open Parenthesis in Function Calls

Avoid placing whitespace before the open parenthesis in function calls. 

# Correct:
function_call(1)

# Incorrect:
function_call (1)

2.2.6 Whitespace Before Open Parenthesis in Indexing or Slicing

Avoid placing whitespace before the open parenthesis in indexing or slicing operations. 

# Correct:
dictionary['key'] = list[index]

# Incorrect:
dictionary ['key'] = list [index]

2.2.7 Avoid Multiple Spaces for Assignment Operators

Use a single space around assignment (or other) operators.

# Correct:
variable1 = 10
variable2 = 20
long_variable_name = 30

# Incorrect:
variable1   = 10
variable2 =  20
long_variable_name    = 30

2.2.8 Avoid Trailing Whitespace

Refrain from having extra spaces at the end of lines in your code.

2.2.9 Use Spaces Around Binary Operators

Always put a single space on both sides of binary operators to improve code readability. 

# Correct:
result = x + y
is_equal = a == b

# Incorrect:
result = x+y
is_equal = a==b

2.2.10 Consider Adding Whitespace for Different Priority Operators

When using operators with different priorities, consider adding whitespace around operators with lower priorities. 

# Correct:
i = i + 1
count += 1
x = x*2 - 1
height = x*x + y*y
c = (a+b) * (a-b)

# Incorrect:
i=i+1
count+=1
x = x*2-1
height = x*x+y*y
c = (a + b)*(a - b)

2.2.11 Function Annotations

Follow normal colon rules and include spaces around the -> arrow when using function annotations. 

# Correct:
def greet(name: str) -> str:
    return f"Hello, {name}!"

# Incorrect:
def greet(name:str)->str:
    return f"Hello, {name}!"

2.2.12 Use Spaces Around the = Sign for Argument Annotations with Default Values

When combining argument annotations with default values, use spaces around the = sign. 

# Correct:
def configure_network(ip: str = "localhost", port: int = 8080):
    # Configuration logic here

# Incorrect:
def configure_network(ip:str="localhost", port:int=8080):
    # Configuration logic here

2.2.13 No Spaces Around the = Sign for Keyword Arguments or Default Values

Avoid using spaces around the = sign when indicating a keyword argument or specifying a default value for an unannotated function parameter. 

# Correct:
def create_point(x=0, y=0):
    return (x, y)

# Incorrect:
def create_point(x = 0, y = 0):
    return (x, y)

2.2.14 Avoid Compound Statements on a Single Line

Discourage combining multiple statements on a single line or folding long lines, especially for multi-clause statements like if-else, try-finally, loops, or function calls. 

# Correct:
if condition:
    do_something()
result = function_call()

# Incorrect:
if condition: do_something(); result = function_call()

2.2.15 Use Trailing Commas for Clarity with Single-Element Tuples

Trailing commas are required when creating a tuple with only one element. For better clarity, it's recommended to surround the single element tuple with parentheses. 

# Correct:
coordinates = (42,)

# Incorrect:
coordinates = 42,

2.2.16 Use Trailing Commas for Consistency in Multiline Lists

Trailing commas are also useful for maintaining consistency in multiline lists. Even if the list does not require extension, adding trailing commas to all elements can make the code more consistent and easier to modify in the future. 

# Correct:
fruits = [
    'apple',
    'banana',
    'cherry',
]

# Incorrect:
fruits = ['apple', 'banana', 'cherry']

2.2.17 Use Trailing Commas in Function Arguments

Applying trailing commas in function arguments, especially when arguments are listed on separate lines, enhances readability and makes it simpler to add or remove arguments in the future. 

# Correct:
result = some_function(
    arg1,
    arg2,
    arg3,
)

# Incorrect:
result = some_function(
    arg1,
    arg2,
    arg3
)
